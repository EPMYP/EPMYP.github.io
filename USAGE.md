# 使用说明

## 🔐 登录和注册功能

### 功能说明

博客系统已添加登录和注册功能，支持两种模式：

1. **后端API模式**（推荐）
   - 需要配置后端服务器
   - 支持完整的用户认证、邮箱验证、OAuth登录
   - 数据存储在服务器端

2. **本地存储模式**（备用）
   - 无需后端服务器
   - 使用浏览器localStorage存储用户信息
   - 适合快速测试和演示

### 使用方式

#### 未登录状态
- 导航栏显示"登录"和"注册"按钮
- 可以浏览已发布的文章
- 无法创建新文章

#### 登录后
- 导航栏显示用户名和下拉菜单
- 可以创建、编辑、删除文章
- 可以配置文章赞赏功能
- 点击用户名可退出登录

### 登录方式

1. **邮箱密码登录**
   - 点击"登录"按钮
   - 输入邮箱和密码
   - 点击"登录"

2. **注册新账号**
   - 点击"注册"按钮
   - 填写用户名、邮箱、密码
   - 点击"注册"
   - 如果配置了邮箱服务，会发送验证邮件

3. **OAuth登录**（需要配置）
   - GitHub登录
   - Google登录
   - QQ登录
   - 微信登录

### 注意事项

- 登录状态保存在浏览器localStorage中
- 清除浏览器数据会退出登录
- 如果使用本地存储模式，密码以明文存储（仅用于演示）
- 生产环境请务必使用后端API模式

## 📝 文章管理

### 创建文章
1. 登录后点击"创建文章"按钮
2. 填写文章信息
3. 选择内容类型（Markdown/HTML）
4. 可选：设置加密、上传封面、添加标签
5. 使用实时预览功能
6. 点击"保存文章"

### 编辑文章
- 在文章列表或详情页点击"编辑"按钮
- 修改内容后保存

### 删除文章
- 在文章列表点击"删除"按钮
- 确认删除

## 🔍 搜索功能

- 在文章列表页面的搜索框输入关键词
- 支持标题、内容、摘要搜索
- 支持标签筛选

## 💰 赞赏功能

1. 在文章详情页找到"赞赏支持"区域
2. 点击支付方式查看二维码
3. 配置支付二维码需要在编辑文章时设置

## 🚀 GitHub Pages部署

项目已配置GitHub Actions，支持自动构建和部署：

1. 推送代码到GitHub
2. 在仓库Settings → Pages中启用GitHub Actions
3. 等待自动部署完成
4. 访问部署的网站

详细说明请查看 [GITHUB_DEPLOY.md](./GITHUB_DEPLOY.md)
